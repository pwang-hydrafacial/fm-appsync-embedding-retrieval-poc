# Demo: AppSync Semantic Retrieval POC

This is a running POC for semantic search over GraphQL on AWS.
A client sends a plain-English question and gets back ranked text passages —
no vector math exposed externally.

---

## Architecture

```
┌─────────────┐        GraphQL query         ┌──────────────────┐
│  CLI client │ ─────────────────────────────▶│  AWS AppSync     │
│ (local)     │  x-api-key header             │  (GraphQL API)   │
└─────────────┘                               └────────┬─────────┘
                                                       │ Lambda invoke
                                                       ▼
                                              ┌──────────────────┐
                                              │  Lambda resolver │
                                              │  (Python 3.12)   │
                                              └──┬───────────┬───┘
                                                 │           │
                              Bedrock invoke      │           │  GetSecretValue
                                                 ▼           ▼
                                        ┌─────────────┐  ┌──────────────────┐
                                        │   Bedrock   │  │ Secrets Manager  │
                                        │ Titan Embed │  │  (DB password)   │
                                        │    v2       │  └────────┬─────────┘
                                        └──────┬──────┘           │ credentials
                                               │ 1024-dim          │
                                               │ embedding         ▼
                                               └──────▶  ┌──────────────────┐
                                                         │  RDS PostgreSQL  │
                                                         │  + pgvector      │
                                                         │  cosine search   │
                                                         └──────────────────┘
```

Everything is Terraform-managed and destroyable with one command.

## Key Components

### GraphQL Schema

The external API contract. The client never touches a vector — only text in, text out.

```graphql
type Query {
  retrieve(queryText: String!, topK: Int = 5): RetrievalResponse!
}

type RetrievalResponse {
  queryText: String!
  matches: [RetrievalMatch!]!
}

type RetrievalMatch {
  documentId:      String!
  chunkId:         String!
  text:            String!
  similarityScore: Float!
  source:          String
}
```

### Database Schema (RDS PostgreSQL + pgvector)

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE document_chunks (
    chunk_id      TEXT PRIMARY KEY,
    document_id   TEXT NOT NULL,
    text          TEXT NOT NULL,
    source        TEXT,
    embedding     vector(1024)    -- Bedrock Titan Embed v2 output
);
```

Similarity search query at retrieval time:

```sql
SELECT chunk_id, document_id, text, source,
       1 - (embedding <=> CAST($1 AS vector)) AS similarity_score
FROM document_chunks
ORDER BY embedding <=> CAST($1 AS vector)
LIMIT $2;
```

## What's Actually in the Database

Two sample chunks from a call center process document, each stored with
a 1024-dimensional embedding vector generated by Bedrock Titan Embed v2.

```
chunk_id:    chunk-001
document_id: doc-001
text:        Agents should verify customer identity before discussing loan details.
source:      sample-doc-1
embedding:   [-0.048160, -0.001597, -0.010970, ... -0.081508]  (1024 dims total)

chunk_id:    chunk-002
document_id: doc-001
text:        Escalate servicing exceptions to the specialist queue.
source:      sample-doc-1
embedding:   [0.002913, 0.031158, 0.002316, ... -0.027106]  (1024 dims total)
```

Each embedding is generated once at seed time and stored. At query time,
the user's question is embedded live and compared against all stored vectors.

## Prerequisites

You need an AWS profile configured with access to this account and Bedrock
model access enabled for `amazon.titan-embed-text-v2:0` in `us-east-1`.

```bash
export AWS_PROFILE=uiuc-pjwang
export AWS_REGION=us-east-1
```

All commands below assume these are set in your shell.

## How to Run It

### 1. Install dependencies and build Lambda layer

Actual command:
```bash
python3 -m venv .venv
. .venv/bin/activate && pip install boto3 pg8000
mkdir -p build/layer/python
pip install -r app/lambda/requirements.txt --target build/layer/python
cd build/layer && zip -r ../layer.zip python/
```

Via Makefile:
```bash
make bootstrap
```

---

### 2. Provision infrastructure

Actual command:
```bash
terraform -chdir=terraform init
terraform -chdir=terraform plan -out=tfplan
terraform -chdir=terraform apply tfplan
```

Via Makefile:
```bash
make tf-init
make tf-plan
make tf-apply
```

This provisions 28 resources: VPC, RDS, Lambda, AppSync, Secrets Manager,
VPC endpoints, IAM roles. Takes ~7 minutes on first run.

---

### 3. Seed the database

Actual command:
```bash
python3 app/seed/seed_data.py
```

Via Makefile:
```bash
make seed
```

Output:
```
Using AWS_PROFILE=uiuc-pjwang AWS_REGION=us-east-1
Connected to RDS. Setting up schema...
  Embedding chunk-001...
  Embedding chunk-002...
Done. Inserted/updated 2 chunks.
```

---

### 4. Run a query

Actual command:
```bash
python3 app/cli/main.py --query "How should agents handle customer identity?"
```

Via Makefile:
```bash
make query q="How should agents handle customer identity?"
```

## Running Result

```
$ make query q="How should agents handle customer identity?"

Query: How should agents handle customer identity?
Matches (2):

  [1] score=0.5873  source=sample-doc-1
      Agents should verify customer identity before discussing loan details.

  [2] score=0.0316  source=sample-doc-1
      Escalate servicing exceptions to the specialist queue.
```

The first chunk scores **0.587** (high cosine similarity) because it directly
answers the question. The second scores **0.032** — it's from the same document
but semantically unrelated to identity verification.

## Tear Down

Actual command:
```bash
terraform -chdir=terraform destroy -auto-approve
```

Via Makefile:
```bash
make tf-destroy
```

Destroys all 28 AWS resources. No snapshots are kept.
